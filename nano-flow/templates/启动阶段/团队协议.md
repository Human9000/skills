# 团队协议

> **目标：** 在项目开始前，所有人对"我们怎么一起工作"达成一致。不假设"大家都知道"。

---

## 四个约定

### 约定一：沟通

**1. 问题怎么报？**
[ ] 群聊 @人（推荐）。还是 [ ] 私聊 PM？还是 [ ] 其他？

**2. 被 @ 的人多久必须响应？**
[ ] 当天。还是 [ ] 24 小时内？还是 [ ] 其他？

**3. 什么信息在群里说？什么信息私聊？**
- 群里说：[ ]
- 私聊说：[ ]

**4. 兼职成员的特殊规则：**

- 无法参加周会 → [ ] 异步看周会记录，24h 内补充意见

---

### 约定二：决策

**1. 日常技术决策谁做？**
[ ] 当事人自己决定（不需要审批）

**2. 什么级别的决定需要拉人讨论？**
[ ] 影响接口的 / [ ] 影响进度的 / [ ] 影响其他人的 / [ ] 其他

**3. 意见不一致时怎么决定？**
[ ] PM 拍板。还是 [ ] 投票？还是 [ ] 找发起人？

---

### 约定三：冲突

**1. 当两个人有分歧时，最多争论多久必须升级？**
[ ] 30 分钟 / [ ] 半天 / [ ] 1 天

**2. 升级给谁？**
[ ] PM

**3. "我不同意但决定已经做了"——执行吗？**
[ ] 执行。保留意见但不在执行中消极抵抗。
 还是 [ ] 其他？

---

### 约定四：反馈

**1. 反馈在什么渠道给？**
[ ] 当面说 / [ ] 群里说 / [ ] 周会上说

**2. "你做错了"这种反馈，怎么说？**
[ ] 直接说 + 给具体例子。对事不对人。

**3. 正面反馈也要说吗？**
[ ] 要。发现别人做了对的事，说出来。
